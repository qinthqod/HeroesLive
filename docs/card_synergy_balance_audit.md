# 卡牌联动与职业平衡审计

## 本轮发现的根因

1. Web 战斗曾主要依靠中文正则读取卡面，只能稳定处理直接伤害、护盾、治疗、抽牌、燃烧和虚弱。
2. 六职业终结技、资源消耗、条件增伤、召回、复制、机关触发等大量效果没有进入运行时。
3. 真解牌曾统一把卡面第一个数字加 3，并把所有有费用的牌减费 1，导致卡面、精研说明和实际强度严重错位。
4. 手牌上限是 7，但战斗界面此前只显示前 5 张，额外抽牌可能实际进入手牌却不可见。

## 已完成的运行时覆盖

- 剑修：攻击连携、藏锋增伤、剑势消耗、多段剑势增幅、虚弱追击、终结返气。
- 符师：符印附着与行动震荡、术法减费、单枚/全部引爆、燃烧即时结算、燃烧增幅。玄雷敕令无符印时不可空放；雷火连符包含自身段数；赤篆连书按实际抽牌类型返气。
- 丹师：寒热药性、丹毒、低血治疗、调和护盾、丹药回收、热性增伤、阴阳终结。
- 御灵：五类灵兽契约、连续指令、当前灵兽行动、契约协同、群兽终结。
- 偃师：机关部署、机巧成长、机关复位、回合末齐射、拆解、护心镜反击、穿透、全机关触发。
- 契师：魂灯点亮与熄灭、低血护持、回响、弃牌、弃牌堆召回、复制、弃牌种类终结。

## 平衡结论

静态基础牌预算检查使用伤害、护盾、治疗、抽牌和持续状态的折算值进行职业间比较。当前六职业平均预算最大差距为 3.36，低于门禁上限 5。

代表性成型联动实测（对 100 生命目标）：

| 职业 | 测试组合 | 实际伤害 | 额外价值 |
| --- | --- | ---: | --- |
| 剑修 | 5 剑势·万剑归岚 | 33 | 返还灵气 |
| 符师 | 4 符印·玄雷敕令 | 24 | 清空符印，满足条件抽牌 |
| 丹师 | 3 热性·逆炼血丹 | 27 | 低费用爆发，承担自伤 |
| 御灵 | 4 契约·百兽同途 | 16 | 同时抽 2 张牌 |
| 偃师 | 4 机关/6 机巧·天工开物 | 36 | 获得护盾并继续强化机关 |
| 契师 | 4 魂灯·魂火焚身 | 28 | 满足条件恢复生命 |

职业不追求完全相同的单卡伤害：丹师和符师需要时间铺状态，偃师依赖持续机关，御灵用抽牌和护盾补偿较低的即时伤害，契师用生命换取高效率。

## 自动门禁

`npm run check:game` 现在同时检查：

- 60 个基础技能模板和 120 张卡牌均具有稳定 `baseName`；
- 每个基础技能在运行时存在对应规则或联动说明；
- 六个终结技均有独立结算分支；
- 真解牌费用变化必须与精研说明一致；
- 真解目标数值必须真实写入卡面；
- 六职业基础牌预算差距不得超过 5。

## 玩家反馈层

- 条件已经满足的卡牌会显示旧金“联”印与高亮边框。
- 满足条件的卡牌会直接显示“联”印与高亮边框；卡牌单击后立即施放，不再经过选中与二次确认。
- 卡牌结算时会展示伤害、护盾、治疗、抽牌、灵气、职业资源和特殊触发明细。
- 6—7 张手牌会自动收紧扇形布局，全部卡牌保持可见；数字键 1—7 均可施放。

## 浏览器战斗烟测

本地试玩服务运行时执行：

```bash
cd web
npm run smoke:combat
```

脚本会通过真实浏览器分别施放剑势终结、符印引爆、热性爆发、灵兽协同、机关终结、魂灯燃魂和弃牌堆终结，并断言敌人最终生命。
