# 用户体验与导航设计

## 适用场景
当需要设计玩家导航系统、评估用户界面、优化菜单结构或设计新手引导时使用此技能。

## 核心原则

### 以用户为中心的设计（UCD）
- **ISO核心要素**：用户参与全过程、迭代评估、关注完整体验、多学科团队
- **关键问题**：
  - 用户是谁？
  - 任务目标是什么？
  - 用户经验水平如何？
  - 使用环境如何（分心vs专注）？
- **工具**：人物角色、场景、用例

### 注意力管理
- 区分感知（被动）与注意（主动）
- 避免无意视盲：不要一次提供过多感官输入，关键信息需逐步展示

## 路径指示方法

### 1. 地标
- 独特外观
- 位于决策节点
- 数量不宜过多

### 2. 标志性建筑
- 大型、显眼
- 吸引玩家视线（像热狗引诱小狗）
- 定义区域范围

### 3. "产道"
- 线性幽闭空间
- 漏斗形收缩后释放到空旷空间
- 制造紧张与期待

### 4. 结构良好的路径
- 清晰传达起点、路线、终点
- 可以是实体路或负空间路

### 5. 光
- 利用本能（飞蛾扑火）
- 引导玩家看向最亮区域
- 传达情绪（暖色安慰、冷色畏惧）

## 希克定律与选择设计

### 数学公式
```
T = a + b × log₂(n)
```
- T：反应时间
- n：可供选择的选项数量
- a, b：取决于特定条件的常数

### 设计推论
- 选项越多，用户越可能放弃选择
- **菜单结构建议**：少一些顶级选项，每个选项下多一些子选项

### 最佳实践
- **合理选项数量**：3个到6个（不少于3个，不多于6个）
- **例外情况**：当用户心里已有具体偏好时，选项越多越好
- **分组策略**：将大量选项归类组织成有限的几个大类别

## 约束条件
- 避免过度引导破坏探索感（除非迷宫是谜题本身）
- 无序、互不相关的选项会使反应时间趋向线性增长
- 避免形式化理解公式，常识和游戏测试胜过数学公式

## 执行步骤
1. 定义目标用户画像和使用场景
2. 确定关键导航节点和决策点
3. 选择合适的路径指示方法组合
4. 应用希克定律优化选项数量和结构
5. 通过用户测试验证导航效果
6. 迭代调整，平衡引导与探索
