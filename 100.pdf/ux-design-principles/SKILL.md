# UX设计原则

## 何时使用
- 设计用户界面布局
- 设计战斗瞄准系统
- 规划任务流程和过场动画
- 需要维持玩家专注度
- 优化交互元素布局

## 持续注意力管理

### 基础法则
人类持续注意力极限为**7至10分钟**。

### 设计调整

**避免过长内容**：
- 过场动画不超过15分钟
- 阅读段落不超过20分钟
- 单个任务不超过10分钟

**分段设计**：
- 将整个体验分解为7-10分钟长的易消化段落
- 每个段落有明确的目标和节奏
- 段落之间提供休息或转换

### 社交游戏特定设计

**核心体验**：
- 设计为10分钟以内
- 适应碎片化时间

**游戏循环**：
- 持续1至2分钟
- 提供小的、可持续的段落
- 快速反馈和奖励

### 注意力重获措施

**被动措施**：
- 每隔7分钟左右展示新元素
- 显示奖励或成就
- 视觉或听觉变化

**主动措施**：
- 通过计时器检测玩家操作
- 若1至2分钟无操作，显示帮助画面
- 显示新任务或其他内容重新吸引

## 菲兹定律应用

### 核心原理
速度与精度的权衡。移动越快，精度越低。

### 关键参数
- **时间**：到达目标所需时间
- **距离**：起始点到目标中心的距离
- **宽度**：目标的大小

### UI设计应用规则

**布局优化**：
- UI元素应靠近用户可能开始点击的地方
- 减少鼠标移动距离
- 常用功能放在易于到达的位置

**尺寸设计**：
- 常用元素应做得更大
- 易于定位和点击
- 重要按钮增加点击区域

**分组设计**：
- 功能类似的元素组合在一起
- 缩短移动距离
- 逻辑分组提高可用性

### 战斗设计应用规则

**难度设定**：
- 考虑人类能力限制
- 测试最快受试者的极限
- 确保挑战在可达成范围内

**控制设计**：
- 核心循环若同时要求速度和精度
- 必须提供清晰简单的控制
- 避免复杂的操作组合

**目标设计**：
- 重要目标应更大
- 移动目标考虑速度和精度平衡
- 提供瞄准辅助

## 设计检查清单

### 注意力管理
- [ ] 单个任务/段落不超过10分钟
- [ ] 过场动画不超过15分钟
- [ ] 有注意力重获机制
- [ ] 社交游戏循环1-2分钟

### 菲兹定律
- [ ] 常用元素靠近起始位置
- [ ] 常用元素尺寸足够大
- [ ] 相关元素分组布局
- [ ] 战斗难度考虑人类极限
- [ ] 核心控制简单清晰

## 常见问题

### 问题：玩家注意力分散
**解决方案**：
- 缩短内容段落
- 增加视觉变化
- 添加互动元素

### 问题：UI难以操作
**解决方案**：
- 增大点击区域
- 优化布局
- 减少移动距离

### 问题：战斗瞄准困难
**解决方案**：
- 增大目标
- 提供瞄准辅助
- 降低移动速度要求
