# 游戏设计核心方法论

## 触发场景
- 开始新游戏的概念设计
- 定义核心玩法循环
- 构建游戏元素的对象-属性-状态模型
- 细化游戏中的交互元素

## 第一部分：创意生成方法

### 1. 自由思考法
- 使用单词气泡、创意之网
- 无结构发散思维
- 适用：早期概念探索

### 2. 结构思考法
- 流程图：有序结构梳理
- 树形图：层次结构梳理
- 适用：逻辑验证和系统设计

### 3. 身体表现法（身体风暴）
- 通过肢体动作辅助设计
- 适用：角色运动或控制机制设计

## 第二部分：核心游戏循环定义

### 循环组成
核心循环 = 一系列具体的动词（如驾驶、射击、潜入）

### 循环流程
```
玩家行动 → 行动结果 → 玩家反应 → 游戏要求重复行动
```

### 特征检查清单
- [ ] 易于理解和操作
- [ ] 令人享受
- [ ] 提供直接反馈
- [ ] 具有适应不同场景的灵活性
- [ ] 具有扩展性（可加入额外动作）
- [ ] 能与其他动作结合
- [ ] 能进化以支持其他循环

### 案例参考
- 《超级马里奥兄弟》：核心循环是"跳跃"，贯穿始终并结合不同挑战
- 《光晕》："快感30秒"，围绕武器顺序、手雷、肉搏、恢复组成的系统化循环

## 第三部分：对象-属性-状态建模

### 步骤1：分解游戏生态系统
游戏 = 游戏空间 + 交互对象 + 玩家输入

### 步骤2：定义对象
识别游戏中的"名词"元素：
- 玩家角色、敌人、物品、障碍物
- 对象可简单（石头）也可复杂（Boss）

### 步骤3：定义属性
为每个对象分配可变或可操控的变量。

**示例：陶罐对象属性**
| 属性 | 值 |
|------|-----|
| 尺寸 | 0.5米×0.5米 |
| 动画 | 空闲状态 |
| 是否可被攻击 | 是 |
| 生命值 | 1 |
| 影响因素 | 所有类型攻击 |

**掉落率配置**：
- 0.5秒延迟
- 15%：1个额外生命值
- 8%：1个金币
- 2%：一袋10个金币
- 75%：没有奖励

### 步骤4：定义状态
确定对象表达属性的方式及触发条件。

**示例：陶罐状态**
| 状态 | 触发条件 | 表现 |
|------|----------|------|
| 活着 | 默认 | 基本存在 |
| 死亡 | 攻击值>1 | 破碎动画，3秒后消失 |

### 步骤5：连锁反应设计
考虑一个对象状态改变对其他对象的影响：
- 引发火灾
- 敌人复活
- 环境变化
